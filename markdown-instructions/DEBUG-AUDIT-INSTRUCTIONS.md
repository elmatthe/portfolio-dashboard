### 6.1 ACB Calculation Accuracy

1. Manually identify 3–5 holdings that have had both BUY and SELL transactions

   - ACB increases on each buy: `new_acb = (old_acb + (shares_bought × price_per_share + commission)) / total_shares`

   - For USD stocks: purchase price must be converted to CAD at the exchange rate on the transaction date
3. Compare your manual result against the value returned by `GET /api/acb`
4. Log PASS/FAIL for each holding, with actual vs. expected values

### 6.2 Capital Gains Accuracy

For each closed position in the example data:

```
Realized Gain/Loss = (Proceeds of Disposition) - (ACB of shares sold) - (Commission)
```

### 6.3 Currency Conversion Accuracy

Verify the exchange rate used for historical transactions:

- The app should use the actual Bank of Canada exchange rate on the transaction date, NOT today's rate
- A USD purchase in January 2024 should use the Jan 2024 USD/CAD rate for ACB purposes
- Log whether historical exchange rates are stored, fetched on demand, or incorrectly using spot rates

### 6.4 TFSA Contribution Room

Verify the TFSA room calculation:

```
Expected 2024 TFSA room = cumulative_room_since_2009 - total_contributions + total_withdrawals_from_prior_year
```
